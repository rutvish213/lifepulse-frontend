Your health data is sacred. Here's how we protect it — at every layer.

Protection

Multi-Layer Security Architecture

Enterprise-grade security protecting over 12,400 donors and 180 hospitals

AES-256 Encryption

All sensitive data — medical records, personal details, blood history — is encrypted using AES-256 standard at rest and TLS 1.3 in transit.

At Rest In Transit
JWT Authentication

JSON Web Tokens with short expiry windows (15 min access / 7 day refresh). Tokens are signed with RS256 and validated on every request.

RS256 Signed Auto Refresh
OTP Verification

All critical actions (registration, blood requests, emergency SOS) require OTP verification via SMS and/or email. OTPs expire in 5 minutes.

Role-Based Access Control

Four distinct roles — Donor, Patient, Hospital, Admin — each with strictly scoped permissions. No cross-role data access without explicit authorization.

4 Roles Least Privilege
Intrusion Detection

Real-time monitoring with automated rate limiting, IP blocking after 5 failed logins, and 24/7 security alerts to our operations team.

Rate Limiting IP Blocking
Secure Database

MongoDB with field-level encryption, daily encrypted backups to AWS S3, point-in-time recovery, and zero-trust network architecture.

Field Encryption Daily Backups

Security Layers in Detail

Network Layer — TLS 1.3 + HTTPS Strict

All communication between client and server uses TLS 1.3 encryption. HSTS headers enforce HTTPS-only connections. Certificate pinning prevents man-in-the-middle attacks.

Application Layer — Input Validation + Helmet.js

All user inputs are sanitized and validated. Express Helmet.js sets security headers — CSP, X-Frame-Options, X-XSS-Protection. SQL/NoSQL injection protection via parameterized queries.

Authentication Layer — JWT + OTP + bcrypt

Passwords hashed with bcrypt (cost factor 12). JWT tokens signed with RS256. OTP verification for registration and critical actions. Refresh token rotation on every use.

Data Layer — AES-256 + Field-Level Encryption

Sensitive fields (Aadhaar, blood records, medical notes) encrypted at field level before storage. Encryption keys stored in AWS KMS, rotated every 90 days.
